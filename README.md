# 직원 이직 분석 · 작은 데이터에서 "진짜 원인"을 어디까지 말할 수 있나

IBM HR Analytics의 직원 1,470명 데이터로 이직의 독립적 요인을 분리한 프로젝트입니다. 다만 핵심은 예측 정확도가 아니라 — **데이터가 작을 때 어떤 도구를 골라야 하는가, 그리고 그 결과를 어디까지 믿어도 되는가**입니다.

> **Stack** · SQLite + SQL · statsmodels(로지스틱 회귀) · scipy(가정 검정) · VIF · 교차검증
> **Data** · [IBM HR Analytics Employee Attrition (Kaggle)](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset) — 1,470명, IBM이 만든 가상 데이터셋

---

## 프로젝트 배경 · 접근 방식

이 프로젝트는 *"또 모델을 하나 돌리자"* 에서 출발하지 않았습니다. 데이터가 **1,470건뿐**이라는 사실에서 출발했습니다.

앞선 두 프로젝트([olist](https://github.com/chamin1010a-droid/olist-delivery-analysis) 9.9만 건, [hospital](https://github.com/chamin1010a-droid/hospital-readmission-analysis) 9.9만 건)에서는 LightGBM이 자연스러운 선택이었습니다. 하지만 1,470건에 블랙박스 ML을 들이대면 두 가지가 무너집니다 — 과적합하기 쉽고, 무엇보다 *"왜 이직하는가"*를 해석할 수 없습니다. 작은 데이터에서 정작 가치 있는 건 예측이 아니라 **해석과 추론**입니다.

그래서 의도적으로 **고전 통계(로지스틱 회귀)**를 골랐습니다. 이 선택 자체가 이 프로젝트의 메시지입니다 — *ML을 쓸 줄 아는 것*보다, **언제 쓰지 *않을지*를 판단하는 것**. 그리고 고전 모델을 골랐다고 끝이 아니라, 그 결과의 신뢰성(공선성·가정·과적합)까지 끝까지 따져 물었습니다.

이 분석은 `문제 정의 → 원인 분리 → (도구 판단) → 검증 → 정책` 흐름을 공유하는 [분석 시리즈](#분석-시리즈)의 **Part 3**입니다.

## 핵심 결과 한눈에

| 발견 | 내용 |
|------|------|
| **가장 강한 위험요인은 야근·잦은출장** | 다른 22개 변수를 통제해도 야근 **OR 6.1**, 잦은출장 **OR 6.8** (p<0.001) |
| **"보상은 영향 없음"은 착각** | 월급·직급은 상관 0.95·VIF 11의 심각한 공선성 — 한쪽을 빼면 *즉시* 유의해짐 |
| **직관적 결과의 함정** | 재직기간이 통제 시 "오래 다닐수록 이직↑"(OR 1.70)으로 뒤집힘 — 단독으론 무관(OR 1.09) |
| **모델 신뢰도** | AUC 0.851, 23개 중 20개 유의, 교차검증상 과적합 없음 |
| **가장 정직한 한계** | 월급·직급·경력은 공선이라 *진짜 원인을 데이터로 구분할 수 없다* |

## 분석 흐름 — 발견이 다음 질문을 불렀다

정해진 단계가 아니라, **한 발견이 다음 질문을 강제하는 식**으로 좁혀졌습니다.

### 1. 작은 데이터다 → 도구부터 고른다

1,470명, 이직 237명(16.1%). 이 크기에 ML을 쓰면 과적합 + 해석 불가. 목표를 *예측*이 아니라 *해석*으로 두고, 고전 통계를 택했습니다. (도구 선택이 결론의 절반입니다.)

### 2. 개별 지표는 뻔하고, 조합은 표본이 부족하다 → 모순

EDA만으로도 신호는 뚜렷했습니다.

| 요인 | 해당 집단 이직률 | 비교 |
|------|:---:|------|
| 야근 | **30.5%** | 비야근 10.4% (약 3배) |
| 잦은 출장 | **24.9%** | 출장 안 함 8.0% |
| 미혼 | **25.5%** | 이혼 10.1% |
| 직무: Sales Representative | **39.8%** | 전체 평균 16.1% |

그런데 *"야근하면 이직률 높으니 챙기자"*는 너무 뻔합니다. 의미는 **조합**("야근 + 미혼 + 저연차")에 있는데, 조합으로 쪼개면 해당자가 30명도 안 돼 통계가 무너집니다. 이 모순이 다음 도구를 불렀습니다.

### 3. 로지스틱 회귀 — 데이터를 쪼개지 않고 통제한다

교차표는 데이터를 실제로 쪼개지만(7명 vs 12명), 로지스틱 회귀는 **1,470명 전체를 한 번에** 써서 각 변수의 *다른 변수를 통제한 독립적 효과*를 오즈비로 분리합니다.

> 모델: **AUC 0.851 · Pseudo R² 0.293 · 23개 변수 중 20개 유의(p<0.05)**

| 요인 (다른 22개 통제) | 오즈비 | p | 방향 |
|------|:---:|:---:|:---:|
| 잦은 출장 (vs 안 함) | **6.80** | <0.001 | 위험 |
| 야근 | **6.06** | <0.001 | 위험 |
| 가끔 출장 (vs 안 함) | 2.93 | 0.004 | 위험 |
| 승진 후 경과 기간 | 1.77 | <0.001 | 위험 |
| 이전 직장 수 | 1.57 | <0.001 | 위험 |
| 통근 거리 | 1.41 | <0.001 | 위험 |
| 현 직무 근속 | 0.61 | 0.002 | 보호 |
| 현 매니저와 근속 | 0.62 | 0.003 | 보호 |

야근·출장이 다른 모든 조건을 통제하고도 6배대로 압도적이고, 지금 직무·지금 상사와 오래 함께한 사람은 덜 떠납니다.

### 4. "그래서 다 믿어도 되나?" → 결과를 의심한다

좋은 오즈비가 나왔다고 끝이 아닙니다. 세 가지를 검증했습니다.

**(가) 다중공선성 — 문제 발견**

| 변수 | VIF | 판정 |
|------|:---:|:---:|
| job_level (직급) | **11.20** | 심각 |
| monthly_income (월급) | **10.83** | 심각 |
| total_working_years (총경력) | 4.77 | 경계 |
| years_at_company (재직기간) | 4.57 | 경계 |
| 나머지 19개 | 1.01 ~ 2.78 | 양호 |

직급과 월급의 상관이 **0.95** — 거의 같은 정보입니다. 전체 모델에서 "월급·직급이 유의하지 않다"고 나온 건 *영향이 없어서*가 아니라 *서로 정보를 희석*했기 때문입니다. 넣고 빼며 확인해 보면:

| 모델 | monthly_income | job_level | total_working_years |
|------|:---:|:---:|:---:|
| 셋 다 넣기 | 0.74 (p=.32) | 0.94 (p=.82) | 0.57 (p=.01) |
| 직급 빼기 | **0.69 (p=.03)** | — | 0.56 (p=.007) |
| 월급 빼기 | — | **0.75 (p=.05)** | 0.56 (p=.007) |
| 월급만 | **0.56 (p=.0001)** | — | — |

→ 한쪽을 빼는 순간 다른 쪽이 살아납니다. "둘 다 무의미"는 착시였습니다.

**(나) 부호 반전** — 재직기간(years_at_company)은 전체 모델에선 "오래 다닐수록 이직↑"으로 나오지만, 단독으로 넣으면 무관해집니다. 다른 근속 변수들을 통제하며 생긴 *허상*입니다.

| 모델 | years_at_company OR | p |
|------|:---:|:---:|
| 근속 변수 4개 전부 | **1.70** (위험?) | 0.02 |
| 단독 | 1.09 (무관) | 0.55 |

**(다) 검정 가정** — point-biserial 상관은 정규성·등분산을 전제하는데, 10개 연속변수가 *전부* 정규분포가 아니었습니다(Shapiro-Wilk p<0.001). 그래서 가정이 필요 없는 **비모수 검정(Mann-Whitney U)**으로 다시 확인해 결론이 흔들리지 않음을 봤습니다.

### 5. 가장 정직한 결론 — 어디까지는 못 말한다

월급·직급·총경력은 서로 강하게 얽혀 있어, *셋 중 무엇이 진짜 원인인지는 이 데이터로 구분할 수 없습니다.* 통제해서 하나만 살아남았다고 그게 원인이라 단정하면 거짓입니다. 관측 데이터의 한계이고, 이걸 인정하는 것까지가 분석입니다.

> 정리하면: *작은 데이터라 통계를 골랐고 → 개별/조합의 모순을 로지스틱 회귀로 풀었지만 → 공선성·부호반전·가정 위반을 발견해 → "강건하게 말할 수 있는 것"과 "말할 수 없는 것"을 갈랐다.*

## 결론 · 제안

### 강건하게 손댈 수 있는 레버
**야근과 잦은출장**은 공선성 문제 없이, 다른 모든 조건을 통제해도 이직 확률을 6배 안팎으로 높입니다. 게다가 회사가 *통제 가능한* 요인입니다 — 업무 분배·출장 빈도 조정, 야근 집중 직무(Sales Representative 등) 우선 관리가 가장 확실한 개입 지점입니다.

### 조심해야 할 해석
**보상(월급·직급)을 "이직과 무관"으로 읽으면 안 됩니다.** 공선성으로 효과가 희석됐을 뿐, 단독으로는 강하게 유의합니다. "연봉은 의미 없다"는 잘못된 정책 결론으로 이어질 수 있는 대표적 함정입니다.

### 한계 — 가장 중요한 단서
이 데이터는 **관측 데이터**이고 변수들이 서로 얽혀 있어, 결과는 *인과가 아니라 연관*입니다. "야근을 줄이면 이직이 준다"를 진짜로 검증하려면 개입 전후 비교나 준실험 설계가 필요합니다. 진짜 원인 분리는 더 많은·다른 데이터의 몫입니다.

## 재현 방법

```bash
# 1. 의존성 설치
pip install pandas numpy scikit-learn statsmodels scipy

# 2. 데이터 준비 — 위 Kaggle 링크에서 CSV를 받아 data/ 에 배치 (저장소 미포함)

# 3. CSV → SQLite 변환 + 정제
python setup_db.py

# 4. 로지스틱 회귀 → 검증 순서로 실행
python logistic_regression.py     # 오즈비로 독립 요인 분리
python check_assumptions.py       # 정규성·등분산·비모수 검정
python check_collinearity.py      # 월급/직급 공선성 실험
python check_years_collinearity.py # 근속 변수 부호 반전 추적
python validate_model.py          # VIF · 교차검증 · 변수 축소 비교
```

## 분석 시리즈

동일한 **`문제 정의 → 원인 분리 → (도구 판단) → 검증 → 정책`** 접근을 도메인·데이터 규모를 바꿔 적용한 연작입니다.

| | 프로젝트 | 도메인 | 차별 포인트 |
|---|---------|--------|------------|
| **Part 1** | [Olist 배송 분석](https://github.com/chamin1010a-droid/olist-delivery-analysis) | 이커머스 | 지연 원인을 *판매자 vs 물류*로 분리, 비대칭 비용 → 분위회귀 |
| **Part 2** | [병원 재입원 분석](https://github.com/chamin1010a-droid/hospital-readmission-analysis) | 헬스케어 | *잔차·SHAP*로 중증도 vs 부주의 분리, 음의 결과를 정직하게 |
| **Part 3** | **직원 이직 분석** *(현재 저장소)* | HR | *데이터가 작아 ML 대신 고전 통계*를 택한 도구 판단 자체가 분석 |

> Part 1·2가 "큰 데이터 + ML로 원인을 분리"였다면, Part 3은 **"작은 데이터에선 무엇을 *하지 않을지*"**를 보여줍니다. 셋 모두 좋아 보이는 결과를 한 번 더 의심하는 검증으로 닫힙니다.
